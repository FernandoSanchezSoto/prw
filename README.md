Descripción del proyecto.


SOSANIMALES es una aplicación que tiene como fin la búsqueda de animales desaparecidos.

Habrán dos tipos de usuarios sin contar el administrador, el usuario principal es aquel que se registra mediante nuestro formato de registro, tiene el privilegio de poder almacenar tantos animales como quiera y generar un aviso por animal, podrá editar los mismos y también eliminarlos.

Por otro lado están los usuarios que no tienen registro hecho, estos solo podrán ver los avisos, no podrán ni generar avisos ni registrar a sus animales en la aplicación.

Mediante un formulario se recogerán los datos del animal, del contacto extra que agregue el usuario y del aviso.


Desarrollo del proyecto.

Definición de requisitos.
Se debe crear un prototipo de página web o aplicación web.

Este prototipo debe contener al menos la siguientes tecnologías:
HTML5.
CSS, SASS, etc.
PHP, JAVA, etc.
MySQL.
Javascript.
Postman
vsCode


Análisis y diseño.
Temática del proyecto.
Temática libre.
Sin límite de páginas web.

Diseño del proyecto.
Prohibido el uso de templates web.
Prohibido Code First, Database First y Model First.
El proyecto debe ser Responsive Web.
Se pueden utilizar Frameworks de diseño.

PHP y MYSQL
Modelo de Base de datos (E/R).
Modelo de Clases de datos (UML).
Implementación CRUD.
Control de errores.
Validación de datos.
Acceso mediante login.
Securización mediante token.
Securización mediante SQL-INJECTION.

Javascript
Control de formularios, validaciones obligatorias de formularios, campos, etc.
